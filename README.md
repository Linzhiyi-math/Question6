# MNIST 手写数字分类：SGD vs 自然梯度（NG-KFAC）对比

##作业说明
本作业在 MNIST 手写数字数据集上实现并比较两种优化算法：**随机梯度下降（SGD）** 与 **自然梯度下降（NG，基于 KFAC 近似）**，仅依赖 `numpy` 与 Python 标准库，无需深度学习框架。

---

## 网络结构

```
输入层 (784)  →  隐藏层 (128, ReLU)  →  输出层 (10, Softmax)
```

- **输入**：28×28 像素图像，展平为 784 维向量，归一化至 [0, 1]
- **隐藏层**：128 个神经元，ReLU 激活，He 初始化
- **输出层**：10 个神经元（对应 0–9），Softmax 输出概率
- **损失函数**：交叉熵损失

---

## 优化算法

### SGD（随机梯度下降）

$$\theta \leftarrow \theta - \eta \nabla L$$

直接沿负梯度方向更新参数，默认学习率 `lr=0.08`。

### NG-KFAC（自然梯度 + Kronecker 分解近似）

$$\theta \leftarrow \theta - \eta \cdot F^{-1} \nabla L$$

利用 Fisher 信息矩阵 $F$ 对梯度进行曲率校正。Fisher 矩阵采用 Kronecker 分解近似：

$$F^l \approx A^l \otimes G^l$$

其中：
- $A^l = \mathbb{E}[a^{l-1}(a^{l-1})^\top]$：输入激活的协方差矩阵
- $G^l = \mathbb{E}[\delta^l(\delta^l)^\top]$：输出梯度的协方差矩阵

默认学习率 `lr=0.01`（Fisher 矩阵已做尺度调整，无需大学习率）。

## 算法原理

### 为什么使用自然梯度？

普通 SGD 在欧氏空间中做梯度下降，未考虑参数空间的曲率。自然梯度通过 Fisher 信息矩阵将参数更新投影到概率分布的 Riemann 空间，使更新方向与损失曲面更为契合，理论上收敛更快、对学习率不敏感。

### KFAC 近似的意义

直接计算 Fisher 矩阵逆的代价为 $O(P^3)$（$P$ 为参数量），不可实用。KFAC 将每层的 Fisher 矩阵分解为两个小矩阵的 Kronecker 积，使求逆代价降至 $O(n^3 + m^3)$（$n, m$ 为输入输出维度），大幅降低计算量。
